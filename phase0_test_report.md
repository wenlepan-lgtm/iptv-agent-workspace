## 问题分析

### 1. 句首严重丢失
- "床前" 完全未识别
- "明月光" 未识别

### 2. 句子被截断
- 第一遍只识别到半句
- 第二遍只识别到后半部分

### 3. 识别错误
- "今夜 丝 还钱" - 完全错误
- "越 正 里 投资 不 下" - 完全错误
- "这 一头 说 头" - 语音混淆

## 可能原因

### 1. VAD 阈值问题
当前 VAD 阈值 80.0 可能导致：
- 句首音量较低时被截断
- 语音起始被延迟检测

### 2. 缓冲区溢出
日志中可能存在 AudioRecord 缓冲区溢出，导致音频数据丢失

### 3. 语速与停顿
用户朗读时句中停顿可能触发 VAD 的 VOICE_END

## 优化建议

### 1. 降低 VAD 阈值
```
asr.vad.threshold=50.0  (从 80.0 降低到 50.0)
```

### 2. 增加静音时长
```
asr.vad.silence.duration.ms=1500  (从 1200ms 增加到 1500ms)
```

### 3. 增加 VAD hangover
```
asr.vad.hangover.frames=6  (从 4 增加到 6)
```
