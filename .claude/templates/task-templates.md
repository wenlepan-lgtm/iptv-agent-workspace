# 任务提示词模板

> 使用这些模板来规范你的任务描述，提高 Claude 的响应质量。

---

## Bug 修复任务模板

```
任务类型：Bug 修复

目标：
修复 XXX 问题，不改变现有架构，不新增依赖，不做无关重构。

强约束：
1. 先阅读相关调用链和状态流，再修改
2. 优先最小改动
3. 涉及框架/SDK行为，先参考官方文档
4. 输出必须包含根因、影响面、风险点、验证步骤
5. 若无法确认根因，先给出候选根因和排查顺序，不要直接乱改

已知现象：
- 现象1: ...
- 现象2: ...

复现步骤：
1. ...
2. ...

怀疑范围：
- 模块A: ...
- 模块B: ...

验收标准：
- [ ] 问题不再复现
- [ ] 无新增 bug
- [ ] 相关功能正常
```

## 功能开发任务模板

```
任务类型：功能开发

目标：
实现 XXX 功能，复用现有代码，不新增不必要的抽象。

强约束：
1. 优先复用现有模块，不重复造轮子
2. 不新增 manager/wrapper/helper，除非证明必要（3处以上复用）
3. 不引入新依赖，除非现有方案不可行
4. 保持代码风格一致

功能描述：
[详细描述功能需求]

技术约束：
- 约束1: ...
- 约束2: ...

参考实现：
[如有参考，列出位置]

验收标准：
- [ ] 功能正常工作
- [ ] 无性能问题
- [ ] 无兼容性问题
- [ ] 代码通过审查
```

## 代码审查任务模板

```
任务类型：代码审查

目标：
审查 XXX 变更，检查架构影响、潜在问题、回归风险。

审查重点：
1. 模块边界是否被破坏
2. 是否有过度设计
3. 是否有潜在 bug
4. 是否有性能问题
5. 是否有安全风险

变更范围：
- 文件1: ...
- 文件2: ...

关注点：
- 关注点1: ...
- 关注点2: ...
```

## 重构任务模板（需谨慎）

```
任务类型：重构

目标：
重构 XXX 模块，[具体目标]，不改变外部行为。

强约束：
1. 不改变公共接口
2. 不改变外部行为
3. 必须有测试覆盖
4. 分步骤进行，每步可回滚

重构原因：
[说明为什么必须重构]

重构范围：
- 范围1: ...
- 范围2: ...

风险评估：
- 风险1: ... - 缓解措施: ...
- 风险2: ... - 缓解措施: ...

验收标准：
- [ ] 所有测试通过
- [ ] 功能行为不变
- [ ] 性能不下降
```
