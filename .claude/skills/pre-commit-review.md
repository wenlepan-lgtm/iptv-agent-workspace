# pre-commit-review-skill

改完代码后自动复核，确保质量。

## 触发方式
```
/pre-commit-review
```

## 执行时机
- 代码修改完成后
- 提交前
- 合并前

## 检查清单

### 1. 代码质量
- [ ] 代码是否可读？
- [ ] 命名是否准确？
- [ ] 是否有冗余代码？
- [ ] 是否有 TODO/FIXME？

### 2. 逻辑正确性
- [ ] 逻辑是否正确？
- [ ] 边界条件是否处理？
- [ ] 错误处理是否完整？
- [ ] 空值处理是否完整？

### 3. 性能
- [ ] 是否有性能问题？
- [ ] 是否有内存泄漏风险？
- [ ] 是否有资源未释放？

### 4. 安全
- [ ] 是否有安全风险？
- [ ] 敏感数据是否保护？
- [ ] 权限是否合理？

### 5. 兼容性
- [ ] 是否破坏现有接口？
- [ ] 是否影响其他模块？
- [ ] 是否考虑版本兼容？

### 6. 测试
- [ ] 是否有对应测试？
- [ ] 测试是否覆盖边界情况？

## 输出模板

```markdown
## 提交前复核报告

### 变更文件
| 文件 | 变更类型 | 行数 |
|------|---------|------|
| xxx.kt | 修改 | +10/-5 |

### 检查结果

#### ✅ 通过项
1. [项目]

#### ⚠️ 警告项
1. [项目] - 建议: [...]

#### ❌ 问题项
1. [项目] - 必须修复

### 影响面分析
- **直接影响**: [模块列表]
- **间接影响**: [模块列表]
- **风险等级**: 低/中/高

### 验证清单
- [ ] 编译通过
- [ ] 单元测试通过
- [ ] 核心功能验证
- [ ] 无新增 lint 警告

### 回滚方式
```bash
git checkout -- [文件列表]
```

### 结论
✅ 可以提交 / ⚠️ 建议修改后提交 / ❌ 需要修复问题
```

## 自动检查脚本

```bash
# 编译检查
./gradlew assembleDebug

# Lint 检查
./gradlew lint

# 单元测试
./gradlew test

# 检查 TODO/FIXME
grep -r "TODO\|FIXME" app/src/main/java --include="*.kt"
```

## 强制约束
- ✅ 必须列出所有变更文件
- ✅ 必须分析影响面
- ✅ 必须提供回滚方式
- ❌ 不允许跳过复核直接提交
