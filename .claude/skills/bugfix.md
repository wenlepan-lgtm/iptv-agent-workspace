# bugfix-skill

所有 Bug 修复必须走这个流程。

## 触发方式
```
/bugfix
```

## 执行流程

### Phase 1: 信息收集
1. 确认问题现象
2. 获取复现步骤
3. 收集相关日志/错误信息
4. 确认最近相关变更

### Phase 2: 调用链追踪
1. 找到入口点
2. 追踪完整调用链
3. 识别涉及的状态变量
4. 检查异步操作和线程模型

### Phase 3: 根因分析
1. 列出 2-3 个候选根因
2. 按可能性排序
3. 说明每个根因的证据
4. 给出验证方法

### Phase 4: 方案选择
1. 对比多个修复方案
2. 选择最小改动方案
3. 评估影响范围
4. 列出回归风险

### Phase 5: 实施修复
1. 只做最小必要修改
2. 不顺手重构
3. 保持代码风格一致

### Phase 6: 验证
1. 验证修复有效
2. 验证无回归
3. 更新相关测试

## 输出模板

```markdown
## Bug 修复报告

### 问题现象
[描述]

### 根因分析
**确认根因**: [描述]
- 位置: [文件:行号]
- 原因: [详细说明]

### 修复方案（最小）
[描述]

### 修改文件
| 文件 | 变更 |
|------|------|
| xxx | 添加 xxx |

### 回归检查点
1. [检查点]

### 验证步骤
1. [步骤]
```

## 强制约束
- ❌ 不允许跳过根因分析直接修改
- ❌ 不允许顺手重构
- ❌ 不允许扩大修改范围
- ✅ 必须列出回归检查点
