## 一、已完成的修复 ✅

### 1.1 GPU 显示格式修复 ✅

**问题**: GPU 显示裸数字 "90000000"，不便于阅读。

**解决**: MetricsCollector.kt 中 `readGpuInfo()` 已修改：
- cur_freq 从 Hz 转换为 MHz 显示
- load 转换为百分比显示

**结果**: ✅ 现在显示为 "GPU: 900MHz" 格式

### 1.2 SwapChain 0x0 黑屏修复 ✅

**问题**: TextureView 在布局完成前可能回调 `onSurfaceTextureAvailable(surface, 0, 0)`，导致 ModelViewer 创建 0x0 SwapChain，一直黑屏。

**解决** (Cursor 修复): FilamentRenderer.kt 中：
- 增加 `surfaceForwardedToModelViewer` 标志
- **仅在** `width > 0 && height > 0` 时，才把 Surface 转发给 ModelViewer
- 若首次是 0x0，等 `onSurfaceTextureSizeChanged` 得到有效尺寸后再转发
- 日志显示 "Surface first valid size: WxH, forwarding to ModelViewer"

**结果**: ✅ Surface 尺寸正确 (465x583)

### 1.3 Entity 跳过逻辑 ✅

当 glb 包含多个 entity 时（如 Blender 默认 Cube），会跳过第一个 entity 避免显示白色方块。

### 1.4 Sparse Accessor 修复 ✅

**问题**: Blender 导出的 glb 包含 24 个 sparse accessors（用于 morph targets），Filament 不支持，导致崩溃。

**解决**: 创建 Python 脚本将 sparse accessors 转换为 dense accessors。

**结果**: ✅ 模型可以加载，不再崩溃

## 七、技术细节

### 7.1 Sparse Accessor 修复

Blender 导出的 glb 中，morph target 数据使用了 sparse accessors 格式来节省空间。
Filament 的 gltfio 不支持 sparse accessors，导致 native 崩溃。

**修复方法**: 使用 Python 脚本将 sparse accessors 扩展为完整的 dense accessors

### 7.2 0x0 SwapChain 修复 (Cursor)

**原因**: TextureView 在布局完成前可能先回调 `onSurfaceTextureAvailable(surface, 0, 0)`。
若把 0x0 的 Surface 交给 ModelViewer，Filament 会创建 0x0 的 SwapChain，导致一直黑屏。

**修复**: 增加 `surfaceForwardedToModelViewer` 标志，仅在有效尺寸时才转发给 ModelViewer。
