# project_brief.md

## 1. 项目名称
本地语音 Agent 项目（工作目录版）

## 2. 项目根路径
`/Users/ala/工作项目/agent`

## 3. 项目目标
建立一套适用于本地开发、代码审核、设备验证的工程化协作机制，使 Claude Code 与 Codex 在同一仓库内基于统一文档协作，而不是依赖聊天记忆。

该机制用于支撑后续 Android / 端侧 / 语音 / ADB 安装 / 盒子验证类任务的稳定推进。

## 4. 当前工作流定位
本仓库当前不是单纯代码仓，也不是单纯文档仓，而是：

- 任务管理入口
- 开发与审核的共同事实源
- 设备验证证据容器
- 项目长期基线承载区

## 5. 协作角色
- Ala：任务提出者、优先级负责人、最终决策者
- Claude Code：开发执行与开发记录维护者
- Codex：代码审核与审核记录维护者

## 6. 基本流程
1. 新建任务卡
2. Claude 开发并写开发记录
3. Codex 审核并写审核记录
4. 如有需要，继续做运行 / ADB / 设备验证
5. Ala 拍板关闭任务

## 7. 长期约束
- 不允许依赖聊天历史作为唯一上下文
- 不允许只改代码不写记录
- 不允许只给审核意见不落盘
- 不允许任务边界无控制扩散
- 不允许把“已验证”写成无证据结论

## 8. 文档优先级
事实优先级如下：
1. 当前代码
2. 当前任务卡
3. 当前项目基线文档
4. 当前开发/审核/验证记录
5. 聊天说明

## 9. 当前初始化说明
当前文件为初始化版本，用于先建立稳定协作骨架。随着项目推进，后续可在不破坏目录结构的前提下扩展内容。
